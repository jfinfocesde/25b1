    Flujo de Trabajo en GitHub: Guía Completa y Tutorial Práctico

    Parte 1: Entendiendo el Flujo de Trabajo en GitHub

    Estructura de Ramas

    1. Ramas Principales:
    2. main (o master): Contiene el código de producción estable

    3. develop: Rama principal de desarrollo

    4. Ramas de Soporte:

    5. feature/*: Para nuevas funcionalidades
    6. hotfix/*: Para correcciones urgentes
    7. release/*: Para preparar nuevas versiones

    Proceso de Desarrollo

    1. Inicio del Desarrollo:
    2. Clonar el repositorio
    3. Crear rama develop desde main

    4. Crear ramas feature desde develop

    5. Desarrollo de Características:

    6. Trabajar en ramas feature
    7. Realizar commits frecuentes

    8. Mantener sincronización con develop

    9. Integración de Cambios:

    10. Crear Pull Requests
    11. Realizar Code Reviews
    12. Resolver conflictos

    13. Merge a develop

    14. Preparación de Releases:

    15. Crear rama release desde develop
    16. Realizar pruebas finales
    17. Merge a main y develop

    Parte 2: Tutorial Práctico - Calculadora Colaborativa

    Configuración Inicial

    Desarrollador 1 (Líder del Proyecto):

    1. Crear el repositorio en GitHub: 
      # Crear y configurar el repositorio local
mkdir calculadora-colaborativa
cd calculadora-colaborativa
git init

# Crear estructura inicial del proyecto
touch README.md
mkdir src

# Commit inicial
git add .
git commit -m "feat: inicialización del proyecto"

# Conectar con GitHub y subir código
git remote add origin [URL_DEL_REPOSITORIO]
git push -u origin main

# Crear y configurar rama develop
git checkout -b develop
git push -u origin develop

    Desarrollador 2 (Colaborador):

    1. Clonar el repositorio: 
      git clone [URL_DEL_REPOSITORIO]
cd calculadora-colaborativa
git checkout develop

    Desarrollo de Funcionalidades

    Desarrollador 1 - Implementar Suma y Resta:

    1. Crear rama de característica: 

      git checkout develop
git pull origin develop
git checkout -b feature/operaciones-basicas

    2. Implementar código: 

      // Calculadora.java
public class Calculadora {
    public double sumar(double a, double b) {
        return a + b;
    }

    public double restar(double a, double b) {
        return a - b;
    }
}

    3. Commit y push: 

      git add .
git commit -m "feat: implementa operaciones de suma y resta"
git push origin feature/operaciones-basicas

    Desarrollador 2 - Implementar Multiplicación y División:

    1. Crear rama de característica: 

      git checkout develop
git pull origin develop
git checkout -b feature/operaciones-avanzadas

    2. Implementar código: 

      // Calculadora.java
public class Calculadora {
    public double multiplicar(double a, double b) {
        return a * b;
    }

    public double dividir(double a, double b) {
        if (b == 0) {
            throw new IllegalArgumentException("No se puede dividir por cero");
        }
        return a / b;
    }
}

    Manejo de Conflictos

    1. Desarrollador 2 - Crear Pull Request:
    2. Ir a GitHub
    3. Crear PR de feature/operaciones-avanzadas a develop

    4. Asignar a Desarrollador 1 como reviewer

    5. Desarrollador 1 - Resolver Conflictos: 

      git checkout develop
git pull origin develop
git checkout feature/operaciones-basicas
git merge develop

    6. Resolver conflictos en Calculadora.java: 

      public class Calculadora {
    public double sumar(double a, double b) {
        return a + b;
    }

    public double restar(double a, double b) {
        return a - b;
    }

    public double multiplicar(double a, double b) {
        return a * b;
    }

    public double dividir(double a, double b) {
        if (b == 0) {
            throw new IllegalArgumentException("No se puede dividir por cero");
        }
        return a / b;
    }
}

    Preparación del Release

    1. Desarrollador 1 - Crear rama release: 

      git checkout develop
git pull origin develop
git checkout -b release/v1.0.0

    2. Ambos Desarrolladores - Pruebas finales

    3. Desarrollador 1 - Merge a main: 

      git checkout main
git merge release/v1.0.0
git tag -a v1.0.0 -m "Primera versión estable"
git push origin main --tags

git checkout develop
git merge release/v1.0.0
git push origin develop

    Buenas Prácticas Implementadas

    1. Comunicación:
    2. Usar mensajes de commit descriptivos
    3. Documentar cambios en PRs

    4. Mantener discusiones en los PRs

    5. Gestión de Código:

    6. Revisiones de código obligatorias
    7. No hacer push directo a main/develop

    8. Mantener ramas actualizadas

    9. Resolución de Conflictos:

    10. Resolver conflictos localmente
    11. Probar después de resolver conflictos
    12. Comunicar cambios significativos

    Este tutorial práctico demuestra un flujo de trabajo real en GitHub, incluyendo todas las interacciones posibles entre dos desarrolladores, desde la configuración inicial hasta el lanzamiento de una versión estable.
